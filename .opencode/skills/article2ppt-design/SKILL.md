# 文章转PPT设计稿技能

本技能用于将一篇博客文章转为PPT设计稿（用yaml自然语言表示），方便用户进行修改。

## 输入参数

用户会提供以下之一：

- 博文 ID（如 `AICodingGetStartedByOCDS`）
- 博文标题（如 `AI Coding入门：以OpenCode+DeepSeek为例开始`）

并可能提供以下参数：

- 输出位置（必须以.yaml结尾才算）

## PPT设计稿格式

PPT设计稿是一个yaml列表，列表中有多个对象。

### 标题页

```yaml
- id: title_page（必须是这个取值）
  type: title_page（必须是这个取值）
  content: <标题页的content必须是博文标题>
```

### 正文页

```yaml
- id: <可自行决定。但应全局唯一>
  type: content_page（必须是这个取值）
  title: <本页的标题>
  content: <多行文字，本页的内容>
  advice: <给PPT设计者的建议>
```
#### 参数说明：id

id参数可以根据当前页面的内容（尤其是标题）指定，使用英文、数字、连接符-命名。

例如，如果有个标题为“例题”，你可以写`example-problem-01`（因为例题通常有好几页）。

#### 参数说明：title

请根据待转换的文章内容决定小标题。

#### 参数说明：content

content是一页PPT的主要文字内容。考虑到空间大小，一页PPT最多有10行文字（带图片的最多5行文字）。考虑到自动换行，长于30字的文字，行数为文字长度/30向上取整。

#### 参数说明：advice

advice是给PPT创作者（用户）的建议。你可以建议某些文字使用动画效果，或者插入某些图片，或者某些图表。

### 结束页

```yaml
- id: end_page（必须是这个取值）
  type: end_page（必须是这个取值）
  content: <一般以“谢谢大家”为宜，但如果你认为文章是一篇影响力强的演讲稿，或者“谢谢大家”不宜，你可以决定>
```

## 执行步骤

### 第一步：查找博文

1. 读取 `src/data/index.yaml`
2. 在 `posts` 列表中匹配用户提供的内容：
   - 如果用户提供 ID，匹配 `id` 字段
   - 如果用户提供标题，匹配 `title` 字段（支持模糊匹配）
3. 如果找不到匹配的博文，告知用户并列出所有可用博文
4. 找到后，确认博文的 ID 和标题

### 第二步：读取博文内容

1. 根据博文 ID 读取文件 `src/data/posts/<id>.md`
2. 记录全文内容。

### 第三步：PPT设计稿制作

思考一篇PPT设计稿应该如何制作。

### 第四步：输出

将PPT设计稿的内容以上方格式输出至输出位置（如提供）或控制台（如不提供位置）。

## 内容格式

### 列表

列表请使用：

1. something
2. something

或：

- something
- something

不能使用①之类的非标准文字。

### 数学公式

使用$包裹的Latex。

## 遵循的原则

将演讲稿转化为PPT，绝不是简单的“复制+粘贴”。演讲稿是听觉逻辑（线性叙事），而PPT是视觉逻辑（结构化展示）。

如果你直接把大段文字贴上去，听众就会在“看屏幕”和“听你讲”之间分裂，效果极差。以下是系统性的转化原则与实操要求，建议收藏细读：

### 一、核心总原则（三大纪律）
提示而非脚本：PPT上的文字是给观众的视觉路标，不是给你的提词器。详细内容由你的口述填充。

一页一概念：每张幻灯片只承载一个核心观点，贪多是PPT设计的第一杀手。

视觉优先于文字：人类处理图像的速度比文字快6万倍。能用图表绝不用表格，能用图片绝不用文字段落。

### 二、内容提炼原则（从“听”到“看”的翻译）
这是最关键的一步，你需要对原文做“减法”：

关键词提取法：将每段演讲稿的核心结论提炼为短句（不超过15字）或关键名词。删除所有修饰性形容词、过渡性口语（如“首先我想说的是”）。

3秒原则：观众应在3秒内读完一张PPT上的所有文字。如果读不完，说明信息过载，需要拆分成2~3页。

黄金分段：演讲稿通常有“开场-主体（3个论点）-结尾”。PPT结构建议为：

封面：标题 + 演讲人 + 日期。

目录页：告诉观众今天要讲哪几块（建议不超过5点）。

过渡页：在每个大章节之间，插入一张“章节分隔页”（只写章节编号+标题），给观众心理上的“分段”喘息感。

内容页：遵循“金字塔原理”，结论先行。

### 三、逐页转化操作清单（拿来即用）
在做每一页时，问自己这 4 个问题：

如果拿走这页PPT，观众会不会听不明白我的口述？——（如果是，补逻辑图）

这页上有没有一句话长度超过两行？——（如果有，拆成两页或提炼动词）

观众拍这张照片（截图）回去，能看懂核心意思吗？——（如果能，合格）

这一页的标题是否能概括整页内容？（标题必须是“结论句”，如“成本下降20%”，而不是“关于成本的汇报”）
