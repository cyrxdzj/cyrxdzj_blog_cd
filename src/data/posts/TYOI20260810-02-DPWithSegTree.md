### 什么是线段树DP

线段树DP是指利用线段树（部分情形下也可用树状数组，但局限较大）这一数据结构来优化动态规划转移的算法技巧。当DP转移方程中出现"在某个区间内查询最值"或"对某个区间进行统一修改"的形式时，可以用线段树将转移的复杂度从 $O(n)$ 降低到 $O(\log n)$。

单调队列DP适用于转移窗口随 $i$ 单调移动的情形，而线段树DP更加灵活：转移区间可以是任意位置、不必单调滑动，还支持区间修改操作。因此，线段树DP的适用范围更广，代价是多一个 $\log$ 因子。

### 基本思想

常见的线段树DP场景有以下两种：

1. **点更新 + 区间查询**：转移方程为 $dp[i] = \max\{dp[j] + w(j)\} \quad (j \in [L_i, R_i])$。用线段树维护所有已计算的 $dp[j] + w(j)$，每次查询 $[L_i, R_i]$ 的最大值，计算 $dp[i]$ 后将其插入线段树。

2. **区间修改 + 区间查询**：转移过程中需要将某个范围内的决策候选值统一加上一个增量，对应线段树的区间加操作。配合区间查询最小值，即可高效完成转移。

### 例题：[Luogu P3084 【USACO13OPEN】 Photo G](https://www.luogu.com.cn/problem/P3084)

#### 题目描述

有 $N$ 头奶牛（$1 \leq N \leq 200\,000$）排成一队，拍了 $M$ 张照片（$1 \leq M \leq 100\,000$）。每张照片覆盖一个连续区间 $[a_i, b_i]$，且照片中有且仅有一头斑点奶牛。求最多可能有多少头斑点奶牛。若不存在合法方案，输出 $-1$。

#### 状态与DP数组的定义

设 $f[i]$ 表示第 $i$ 头牛强制为斑点牛时，前 $i$ 头牛中斑点牛的最大数量。

为了方便处理边界，添加一头虚拟牛 $N+1$（视为斑点牛），最终答案为 $f[N+1] - 1$。

#### 转移方程

从 $f[j]$ 转移到 $f[i]$（$j < i$）需要满足两个约束：

1. **每个区间至多一头斑点牛**：$j$ 必须小于所有包含 $i$ 的区间左端点。定义 $R[i] = \min\{\text{所有包含 } i \text{ 的区间的左端点}\}$（不存在则 $R[i] = i$），要求 $j < R[i]$。

2. **每个区间至少一头斑点牛**：完全在 $i$ 左侧的区间内都必须已有斑点牛。定义 $L[i] = \max\{\text{所有右端点 } < i \text{ 的区间的左端点}\}$（不存在则 $L[i] = 0$），要求 $L[i] \leq j$。

因此转移方程为：

$$
f[i] = \max_{L[i] \leq j < R[i]} \{ f[j] \} + 1
$$

$L[i]$ 和 $R[i]$ 的预处理方式与单调队列DP文章中的 Photo G 完全一致：先根据区间信息初始化边界数组，然后正推 $L[i]$、倒推 $R[i]$。

#### 线段树优化

转移需要在 $[L[i], R[i]-1]$ 中查询 $f[j]$ 的最大值，然后单点更新 $f[i]$。这正好是线段树的经典应用：建一棵维护区间最大值的线段树，初始时 $f[0] = 0$（虚拟点），其余为 $-\infty$。

$$
f[i] = \text{segTree.query}(L[i],\, R[i]-1) + 1
$$

处理完 $f[i]$ 后执行 $\text{segTree.update}(i,\, f[i])$。

#### 复杂度分析

- **时间复杂度**：预处理 $O(N + M)$，DP 部分每次转移需一次线段树查询和一次更新，均为 $O(\log N)$，总时间 $O(N \log N + M)$。
- **空间复杂度**：$O(N)$。

### 例题：[Luogu P3287 【SCOI2014】 方伯伯的玉米田](https://www.luogu.com.cn/problem/P3287)

#### 题目描述

给定长度为 $N$ 的序列 $a[i]$（$1 \leq N \leq 10^4$，$1 \leq K \leq 500$，$1 \leq a[i] \leq 5000$），最多进行 $K$ 次操作，每次操作可以选择一个区间 $[L, R]$，将区间内所有玉米高度 $+1$。操作结束后可以删除若干玉米，问剩下的玉米构成**单调不下降序列**的最长长度。

#### 关键性质

**每次区间加的右端点一定是 $N$**。证明：若右端点不到 $N$，则从右端点右侧未被区间加覆盖的位置无法继承左侧的长度，将右端点延伸到 $N$ 一定不劣。

因此，对于第 $i$ 个玉米，若被 $j$ 次操作覆盖（$0 \leq j \leq K$），其最终高度为 $a[i] + j$。

#### 状态与DP数组的定义

设 $dp[i][j]$ 表示以第 $i$ 个玉米结尾、该玉米被拔高了 $j$ 次时的最长单调不下降子序列长度。

#### 转移方程

要转移到 $dp[i][j]$，需要在前 $i-1$ 个玉米中找一个结尾玉米 $k$，其拔高次数 $l \leq j$，且最终高度 $a[k] + l \leq a[i] + j$：

$$
dp[i][j] = \max_{k < i,\; l \leq j,\; a[k] + l \leq a[i] + j} \{ dp[k][l] \} + 1
$$

这是一个三维偏序问题（$k < i$、$l \leq j$、$a[k] + l \leq a[i] + j$）。由于 $i$ 天然递增枚举，只需同时维护后两维。

注意：计算 $dp[i][j]$ 时，$j$ 必须从 $K$ 递减到 $0$ 枚举，以避免同一玉米在子序列中出现两次。

#### 二维树状数组优化

将 $(a[k] + l,\; l)$ 看作二维坐标，$dp[k][l]$ 作为该点的值。转移时查询矩形 $[1, a[i] + j] \times [1, j+1]$ 内的最大值，$dp[i][j]$ 计算后插入点 $(a[i] + j,\; j+1)$（树状数组下标从 $1$ 开始，因此第二维偏移 $1$）。

用二维树状数组（Fenwick Tree）维护前缀最大值。第一维为最终高度，上限为 $\max a + K$（约 $5500$）；第二维为操作次数，上限为 $K$（$\leq 500$），内存和效率均可接受。

#### 复杂度分析

- **时间复杂度**：两维树状数组各带一个 $\log$，每次查询和修改需访问约 $\log(H) \cdot \log(K)$ 个节点，总节点访问次数约 $N \cdot K \cdot \log(H) \cdot \log(K) \approx 10^4 \times 500 \times 13 \times 9 \approx 5.85 \times 10^8$。实际二维 BIT 常数极小，且 $K$ 通常远小于上限，可以通过。
- **空间复杂度**：$O(H \cdot K) \approx 5500 \times 501$，约 $2.75 \times 10^6$ 个 `int`。

### 例题：[Luogu P2605 【ZJOI2010】 基站选址](https://www.luogu.com.cn/problem/P2605)

#### 题目描述

有 $n$ 个村庄在一条直线上，第 $i$ 个村庄到第 $1$ 个村庄的距离为 $d_i$（严格递增）。需要建立不超过 $k$ 个通讯基站。在第 $i$ 个村庄建基站的代价为 $c_i$。如果在距离第 $i$ 个村庄 $s_i$ 范围内有基站，则该村庄被覆盖；否则需补偿 $w_i$。求最小总代价。

数据范围：$n \leq 2 \times 10^4$，$k \leq \min(n, 100)$，$1 \leq d_i,s_i \leq 10^9$，$1 \leq c_i,w_i \leq 10^4$。

#### 状态与DP数组的定义

设 $dp[i][j]$ 表示仅考虑前 $i$ 个村庄，总共建立 $j$ 个基站，且最后一个基站建在 $i$ 处的最小总代价。

为了统一答案计算，在末尾添加一个虚拟村庄 $n+1$（距离无限大、建设费 $0$、补偿费 $+\infty$），并将 $k$ 增加 $1$（为虚拟村庄预留一个基站名额）。最终答案为 $\min_{j=1}^{k} dp[n+1][j]$，其中 $k$ 已自增，对应放置不超过原 $k$ 个基站的最小总代价（允许一个基站都不建，此时仅虚拟村有基站）。

#### 转移方程

$$
dp[i][j] = \min_{j-1 \leq p \leq i-1} \big( dp[p][j-1] + \text{cost}(p, i) \big) + c_i
$$

其中 $\text{cost}(p, i)$ 表示区间 $[p, i]$ 中既不能被 $p$ 处基站覆盖、也不能被 $i$ 处基站覆盖的村庄的补偿费之和。

#### 线段树优化

用滚动数组消去第二维。核心思路：外层循环基站数量 $j$，内层从左到右扫描 $i$。枚举 $i$ 的过程中，用线段树维护 "$dp[p] + \text{cost}(p, i)$" 的最小值。注意 $j=1$ 时需单独处理（无上一个基站，直接递推），只有在 $j \geq 2$ 时才使用线段树优化。

预处理每个村庄 $x$ 能被覆盖的基站区间 $[st_x, ed_x]$（在 $d$ 数组上二分查找确定）。用邻接表记录 $ed$ 值为 $x$ 的村庄编号。

当 $i$ 向右移动时，之前某些村庄 $x$（满足 $ed_x = i$）从此再也无法被后面的基站覆盖。对于所有决策点 $k < st_x$，若转移时选择 $k$ 作为上一个基站，则村庄 $x$ 无法被覆盖，需要赔 $w_x$。因此在线段树上给区间 $[1, st_x - 1]$ 加上 $w_x$。

#### 复杂度分析

- **时间复杂度**：外层 $k$ 次迭代，每次迭代 $O(n \log n)$（每个村庄涉及一次查询和若干次区间加），总时间 $O(k n \log n)$。$k \leq 100$，$n \leq 2 \times 10^4$，可以接受。
- **空间复杂度**：$O(n)$。

### 线段树DP的一般套路

1. **识别特征**：转移方程中出现区间查询最值（点更新 + 区间查询）或区间更新最值（区间修改 + 区间查询），且转移区间不一定是单调滑动的窗口。
2. **选择数据结构**：
   - 若只有**点更新 + 区间查询**，且查询是前缀或矩形，优先考虑树状数组（常数小）；
   - 若涉及**区间修改**（如区间加），必须使用带懒标记的线段树。
3. **确定维护内容**：线段树的叶子节点存储"如果选择该下标作为决策点，当前阶段的最小/最大代价"，父节点合并区间最值。
4. **处理转移中的额外代价**：当 $i$ 向右移动时，某些之前不可见的代价可能会变得可见（如 P2605 中村庄超出覆盖范围时需要补偿），通过区间加操作动态更新所有受影响决策点的代价。
5. **滚动数组**：若 DP 有第二维（如基站个数），外循环枚举第二维，内循环用线段树加速转移，每轮结束后用新 DP 值重建线段树。
