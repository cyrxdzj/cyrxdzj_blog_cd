### 什么是单调队列DP

单调队列DP是指利用单调队列这一数据结构来优化动态规划转移的算法技巧。当DP转移方程中出现"在一段滑动区间中取最值"的形式时，可以用单调队列将单次转移的复杂度从 $O(k)$（$k$ 为窗口大小）降低到均摊 $O(1)$

单调队列本身是解决"滑动窗口最值"问题的经典数据结构。当它与DP结合时，我们维护一个队列，队列中存储的是**可能成为最优决策的下标**，队列内元素按某种单调性排列（如队头为最大值），同时确保队列中的下标始终在当前决策允许的范围内。

### 基本思想

设 DP 转移方程为：

$dp[i] = \max\{dp[j] + w(i,j)\} \quad (j \in [LEFT_i,RIGHT_i])$

> $[LEFT_i,RIGHT_i]$ 是滑动窗口。

其中 $w(i,j)$ 可以分离为只与 $i$ 有关的部分和只与 $j$ 有关的部分（即 $w(i,j) = f(i) + g(j)$）。此时方程改写为：

$dp[i] = f(i) + \max\{dp[j] + g(j)\} \quad (j \in [LEFT_i,RIGHT_i])$

于是问题转化为：随着 $i$ 递增，维护一个滑动窗口 $[LEFT_i,RIGHT_i]$ 内的 $\{dp[j] + g(j)\}$ 最大值。这正是单调队列的用武之地。

**核心操作**：
- **队头出队条件**：队头下标 $q[head]$ 已滑出当前窗口左边界（即 $q[head] < i - R$），此时 $q[head]$ 不再能转移到 $i$，将其弹出。
- **队尾入队操作**：在将新下标 $i - L$（即将进入窗口的下标）入队前，不断弹出队尾那些 $dp[q[tail]] + g(q[tail]) \le dp[i-L] + g(i-L)$ 的下标，以保持队列的单调递减性。
- **转移**：取队头元素作为最优决策，计算 $dp[i]$。
>先入队再转移还是先转移再入队取决于题目要求。例如，有时先入队再转移有可能导致还没有计算的当前下标错误地参与dp[i]的计算。总之，需要确保该入队参与计算的值一个不少。

### 例题：[Luogu P1725 琪露诺](https://www.luogu.com.cn/problem/P1725)

#### 题目描述

琪露诺从编号为 $0$ 的格子出发，要跳过编号为 $N$ 的格子（即到达 $> N$ 的位置）。在位置 $i$ 时，她只能跳到区间 $[i + L, i + R]$ 内的格子。每个格子有一个冰冻指数 $a[i]$（可能为负数），当她停留在格子 $i$ 时可获得 $a[i]$。求最大冰冻指数之和。

数据范围：$N \leq 2 \times 10^5$，$1 \leq L \leq R \leq N$，$-10^3 \leq a[i] \leq 10^3$。

#### 状态与DP数组的定义

设 $dp[i]$ 表示跳到格子 $i$ 时能获得的最大冰冻指数之和。目标是在 $i \in [N+1, N+R]$ 中取 $\max\{dp[i]\}$（因为超过 $N$ 就算到达对岸，且最后一步的 $a[i]$ 已无格子可取），实际上只需关心 $i \in [L, N+R]$。

#### 转移方程与单调队列优化

要跳到格子 $i$，上一步 $j$ 必须满足 $j \in [i - R, i - L]$。因此：

$dp[i] = a[i] + \max\{dp[j]\} \quad (j \in [i - R, i - L])$

另外，需将数组开至 $N+R$ 且 $a[N+1..N+R] = 0$。

**单调队列的维护**：
- 遍历 $i$ 从 $1$ 到 $N+R$。如果 $i\ge L$，则将 $i - L$ 作为新的候选下标准备入队。
- **队头出队条件**：$queue[head] + R < i$，即队头的最远跳跃距离到不了 $i$，弹出。
- **队尾入队操作**：如果 $i\ge L$，则设入队下标为 $p = i - L$。当队列非空且 $dp[queue[tail]] \leq dp[p]$ 时，弹掉队尾（新值更大，旧值失去成为最优的潜力）。然后将 $p$ 入队。
- **转移**：$dp[i] = a[i] + dp[queue[head]]$。记得检查队列是否为空，如为空则设 $dp[i]=-\infty$。

#### 复杂度分析

每个下标最多入队一次、出队一次，时间复杂度 $O(N)$，空间复杂度 $O(N)$。

### 例题：[Luogu P2034 选择数字](https://www.luogu.com.cn/problem/P2034)

#### 题目描述

给定一行 $n$ 个非负整数 $a[1] \sim a[n]$。可以从中选择若干个数，但不能有超过 $k$ 个连续的数字被选择。求选出的数字之和的最大值。

数据范围：$n \leq 10^5$，$k \leq n$。

#### 状态与DP数组的定义

正难则反，考虑删除若干个数，使得任意两个被删除的数之间不超过 $k$ 个数（即不存在连续 $k+1$ 个未被删除的数），且删除的数的和最小。

设 $sum[i] = \sum_{t=1}^{i} a[t]$ 为前缀和。设 $dp[i]$ 表示前 $i$ 个数中，强制删掉第 $i$ 个数时的最小删除和。那么被删除的数的和越小，被选中的数的和就越大。最终答案需要在所有可能的"最后一个被删的数"中取最小值，且最后一个被删的数之后（即尾部）不能有超过 $k$ 个连续被选的数，因此最后一个被删的数必须落在 $[n - k, n]$ 内：$ans = sum[n] - \min\limits_{i = n-k}^{n} dp[i]$。

#### 转移方程与单调队列优化

第 $i$ 个数被删除时，前一个被删除的数 $j$ 必须满足 $j \in [\max(0, i-k-1), i - 1]$，否则 $j+1 \sim i-1$ 这些被选的数会超过 $k$ 个。因此：

$dp[i] = a[i] + \min\{dp[j]\} \quad (j \in [\max(0, i-k-1), i - 1])$

其中 $dp[0] = 0$ 表示虚拟的被删起点。

由于 $\min\{dp[j]\}$ 的区间随 $i$ 滑动，用单调递增队列维护 $dp[j]$ 的最小值。

**单调队列的维护**：
- 初始化 $dp[0] = 0$，队列中先放入下标 $0$。
- 遍历 $i$ 从 $1$ 到 $n$，窗口为 $[i - k - 1, i - 1]$。
- **队头出队条件**：$queue[head] < \max(0, i-k-1)$，即队头超出窗口左边界，弹出。
- **转移**：$dp[i] = a[i] + dp[queue[head]]$。
- **队尾入队操作**：将下标 $i$ 作为新候选入队。当队列非空且 $dp[queue[tail]] \geq dp[i]$ 时，队尾被弹掉（维护单调递增）。然后将 $i$ 入队。

#### 复杂度分析

时间复杂度 $O(n)$，空间复杂度 $O(n)$。

### 例题：[Luogu P2569 [SCOI2010] 股票交易](https://www.luogu.com.cn/problem/P2569)

#### 题目描述

lxhgww 预测了未来 $T$ 天内某只股票的走势。第 $i$ 天的买入价为 $AP_i$（每股），卖出价为 $BP_i$（每股，$AP_i \geq BP_i$），当天最多买入 $AS_i$ 股、卖出 $BS_i$ 股。两次交易（买入或卖出）之间必须间隔至少 $W$ 天。初始时有 $0$ 股，资金无限。求 $T$ 天后的最大收益。

数据范围：$T \leq 2000$，$MaxP \leq 2000$，$W \leq T$。

#### 状态与DP数组的定义

设 $dp[i][j]$ 表示第 $i$ 天结束时，持有 $j$ 股股票时的最大收益。

应初始化 $dp[i][j]=-\infty$ 且 $dp[0][0]=0$。

#### 转移方程与单调队列优化

分四类情况讨论：

**1. 第一次买入**：此前从未进行过交易，在第 $i$ 天直接买入 $j$ 股。

$dp[i][j] = -j \times AP_i \quad (0 \leq j \leq AS_i)$

**2. 不操作**：第 $i$ 天不做任何交易，直接继承第 $i-1$ 天的状态。

$dp[i][j] = \max(dp[i][j],\; dp[i-1][j])$

**3. 买入**：在第 $i$ 天买入若干股，从上一次交易日（第 $i-W-1$ 天）转移而来。设第 $i-W-1$ 天持有 $k$ 股，第 $i$ 天买入后持有 $j$ 股（$j \geq k$）。

$dp[i][j] = \max\{dp[i-W-1][k] - (j - k) \times AP_i\} \quad (k \in [j - AS_i,\; j])$

变形为：

$dp[i][j] = -j \times AP_i + \max\{dp[i-W-1][k] + k \times AP_i\} \quad (k \in [j - AS_i,\; j])$

此时需要从滑动窗口 $[j - AS_i,\; j]$ 中取出 $dp[i-W-1][k] + k \times AP_i$ 的最大值。用单调递减队列维护该值。

**单调队列维护买入（正序枚举 $j$ 从 $0$ 到 $MaxP$）**：
- **队头出队条件**：$queue[head] < j - AS_i$，即队头代表的股数太少，即使以最大限额 $AS_i$ 买入，持股数也达不到 $j$，弹出。
- **队尾入队操作**：将 $k = j$ 作为新候选入队。当队列非空且 $dp[i-W-1][queue[tail]] + queue[tail] \times AP_i \leq dp[i-W-1][j] + j \times AP_i$ 时，弹出队尾。然后将 $j$ 入队。
- **转移**：若队列非空，$dp[i][j] = \max(dp[i][j],\; -j \times AP_i + dp[i-W-1][queue[head]] + queue[head] \times AP_i)$。

**4. 卖出**：在第 $i$ 天卖出若干股，从上一次交易日（第 $i-W-1$ 天）转移而来。设第 $i-W-1$ 天持有 $k$ 股，第 $i$ 天卖出后持有 $j$ 股（$j \leq k$）。

$dp[i][j] = \max\{dp[i-W-1][k] + (k - j) \times BP_i\} \quad (k \in [j,\; j + BS_i])$

变形为：

$dp[i][j] = -j \times BP_i + \max\{dp[i-W-1][k] + k \times BP_i\} \quad (k \in [j,\; j + BS_i])$

同样用单调递减队列维护该值，但窗口是 $[j,\; j + BS_i]$，随着 $j$ 减小窗口向左滑动。

> 递减指的是按 $dp[...][k] + k \times AP_i$ 的值单调递减维护，队列中存储的是股数 $k$。

**单调队列维护卖出（倒序枚举 $j$ 从 $MaxP$ 到 $0$）**：
- **队头出队条件**：$queue[head] > j + BS_i$，即队头代表的股数太多，超出窗口右边界，即使以最大限额 $BS_i$ 卖出，持股数也降不到 $j$，弹出。
- **队尾入队操作**：将 $k = j$ 作为新候选入队。当队列非空且 $dp[i-W-1][queue[tail]] + queue[tail] \times BP_i \leq dp[i-W-1][j] + j \times BP_i$ 时，弹出队尾。然后将 $j$ 入队。
- **转移**：若队列非空，$dp[i][j] = \max(dp[i][j],\; -j \times BP_i + dp[i-W-1][queue[head]] + queue[head] \times BP_i)$。

**注意**：$i \leq W$ 时无法从之前的交易日转移（间隔不足 $W$ 天），只需处理情况 1 和情况 2。

#### 复杂度分析

外层循环天数 $T$，内层循环股数 $MaxP$，每个 $j$ 在单调队列中进出各一次，总时间复杂度 $O(T \cdot MaxP)$，空间复杂度可以优化为 $O(MaxP)$（滚动数组），但朴素 $O(T \cdot MaxP)$ 的空间也符合数据范围。

### 单调队列优化DP的一般套路

1. **识别特征**：转移方程中出现形如 $\max\{dp[j] + g(j)\}$ 或 $\min\{dp[j] + g(j)\}$，且 $j$ 的取值范围是随 $i$ 单调移动的滑动窗口；
2. **分离变量**：将方程改写为 $dp[i] = f(i) + \max\{dp[j] + g(j)\}$ 的形式，确保 $\max$ 内部只与 $j$ 有关，确保 $i$ 与 $j$ 的项完全拆分；
3. **确定窗口**：明确 $j$ 的取值范围 $[L(i), R(i)]$，且 $L(i)$ 和 $R(i)$ 随 $i$ 单调不减；
4. **维护单调队列**：
   - 用数组模拟双端队列，$head$ 和 $tail$ 分别指向队首和队尾；
   - 入队前检查队尾：对于 $\max$ 问题，若 $dp[queue[tail]] + g(queue[tail]) \le$ 新值，则弹出队尾（维护递减队列）；对于 $\min$ 问题，若 $dp[queue[tail]] + g(queue[tail]) \ge$ 新值，则弹出队尾（维护递增队列）；
   - 转移前检查队头：若队头元素已滑出窗口，则弹出队头；
   - 取队头作为最优决策进行转移。
5. **边界初始化**：处理 $i$ 较小时无法从窗口中取到有效决策的特殊情况，根据题目要求可在队列中预先放入 $dp[0]$ 对应的下标 $0$。
