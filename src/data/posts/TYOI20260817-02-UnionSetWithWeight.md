### 什么是带权并查集

普通并查集只能判断两个元素是否属于同一集合，但有时我们不仅需要知道元素是否在同一集合，还需要维护元素之间的某种**量化关系**，比如距离、差值、相对位置等。

**带权并查集**在普通并查集的基础上，为每个节点增加了一个**权值**（通常记为 `dis[x]`），用来表示该节点与其父节点之间的某种关系量。通过路径压缩时正确更新权值，我们可以在查询时计算出任意两个节点之间的关系。

### 核心思想

带权并查集维护三个数组：

- `fa[x]`：节点 $x$ 的父节点
- `dis[x]`：节点 $x$ 到其父节点的权值（表示两者之间的关系量）
- `sz[x]`：以 $x$ 为根的集合大小（仅对根节点有效，根据题目需要可选）

进行路径压缩时，`dis[x]` 需要累加更新，使其最终表示节点 $x$ 到根节点的权值：

```cpp
// 查找根节点（带路径压缩，同时更新dis）
int find(int x)
{
    if (fa[x] == x) return x;
    int root = find(fa[x]);
    dis[x] += dis[fa[x]];   // 将到父节点的权值累加到到根节点的权值
    return fa[x] = root;
}
```

合并操作时，根据题目给出的关系，计算连接两个根节点所需的权值偏移量。

### 路径压缩中dis更新的原理

路径压缩是带权并查集最容易出错的地方。假设链为 $x \to p \to q \to root$（$p$ 是 $x$ 的父节点，$q$ 是 $p$ 的父节点，$root$ 是根）：

- 压缩前：`dis[x]` 是 $x$ 到 $p$ 的距离，`dis[p]` 是 $p$ 到 $q$ 的距离
- 当我们递归 `find(p)` 后，$p$ 直接指向 $root$，且 `dis[p]` 更新为 $p$ 到 $root$ 的距离
- 然后更新 `dis[x] += dis[fa[x]]`，即 `dis[x]` 加上 $p$ 到 $root$ 的距离，得到 $x$ 到 $root$ 的总距离

这个过程确保了路径压缩后，每个节点的 `dis` 值都正确反映了它到根节点的距离。

### 例题

#### 1. [Luogu P1196 【NOI2002】 银河英雄传说](https://www.luogu.com.cn/problem/P1196)

对于每艘战舰 $i$，我们维护：

- `fa[i]`：$i$ 的父节点（在队列中排在前面的战舰）
- `dis[i]`：$i$ 到其父节点的距离（即排在 $i$ 前面的战舰数量）
- `sz[i]`：如果 $i$ 是队列的队首（根节点），则 `sz[i]` 表示该队列的战舰总数

**合并操作** `M i j`（将 $i$ 所在队列接在 $j$ 所在队列的尾部）：

1. 找到 $i$ 的根节点 $ri$，$j$ 的根节点 $rj$
2. 将 $ri$ 的父节点设为 $rj$（即 $ri$ 接在 $rj$ 后面）
3. $ri$ 到 $rj$ 的距离 = $j$ 所在队列的总战舰数 = `sz[rj]`
4. 更新 $j$ 所在队列的大小：`sz[rj] += sz[ri]`

**查询操作** `C i j`：

1. 若 `find(i) != find(j)`，说明不在同一队列
2. 否则，它们之间相隔的战舰数量为 `abs(dis[i] - dis[j]) - 1`

因为 `dis[i]` 表示排在第 $i$ 号战舰前面的战舰数量，`dis[j]` 同理，所以两者之间相隔 `abs(dis[i] - dis[j]) - 1` 艘战舰。
