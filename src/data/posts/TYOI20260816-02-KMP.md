### 什么是KMP算法？
KMP算法是一种高效的字符串匹配算法，由Knuth、Morris和Pratt三位科学家提出，时间复杂度为O(n+m)，其中n是文本串长度，m是模式串长度。相比暴力匹配的O(nm)复杂度，KMP通过避免重复匹配大幅提升效率。

### 核心：fail数组
fail数组（也称next数组、前缀函数）是KMP算法的核心，fail[i]表示模式串中前缀[1,i]的最长的、同时是真前缀和真后缀的子串的长度。

#### fail数组的推导
我们以模式串S = "ababcabab"为例，逐步推导fail数组：（本文章中字符串下标从1开始）
| 下标i | 子串 | 最长相等真前缀与真后缀 | fail[i] |
|-------|------|------------------------|---------|
| 1     | a    | 无                     | 0       |
| 2     | ab   | 无                     | 0       |
| 3     | aba  | a                      | 1       |
| 4     | abab | ab                     | 2       |
| 5     | ababc| 无                     | 0       |
| 6     | ababca| a                     | 1       |
| 7     | ababcab| ab                   | 2       |
| 8     | ababcaba| aba                 | 3       |
| 9     | ababcabab| abab               | 4       |

> 注：部分教材采用0-based下标规范，本博客采用1-based下标规范是为了更符合日常计数习惯。

#### fail数组的推导证明
**证明：** $fail[i] = k$ 满足 $S[1,k] = S[i-k+1,i]$，且 $k$ 是满足该条件的最大值
1. 基础情况：当 $i=1$ 时，只有一个字符，没有真前缀和真后缀，$fail[1]=0$ 显然成立
2. 归纳假设：假设对于所有 $j < i$，$fail[j]$ 已经正确计算
3. 归纳步骤：对于 $i$，我们知道 $fail[i-1]=k$，即 $S[1,k] = S[i-k,i-1]$
   - 如果 $S[k+1] == S[i]$，那么显然 $S[1,k+1] = S[i-k,i]$，所以 $fail[i] = k+1$
   - 如果 $S[k+1] \neq S[i]$，那么我们要找更小的可能的 $k'$。由于 $S[1,k] = S[i-k,i-1]$，$S[1,k]$ 的任意相等前后缀也是 $S[i-k,i-1]$ 的后缀，所以可以直接继承作为候选，即 $k' = fail[k]$。重复上述检查，直到 $k=0$，此时 $fail[i]=0$

该过程确保了每次找到的都是最长的相等前后缀长度，证明成立。

### 例题：[Luogu P3375 【模板】KMP字符串匹配](https://www.luogu.com.cn/problem/P3375)

#### 匹配过程
匹配过程利用fail数组，当发生失配时，不需要回溯文本串指针，只需要将模式串指针移动到 $fail[j]$ 的位置继续匹配（以下采用与代码一致的1-based下标描述）：
1. 初始化 $i=1$（文本串指针），$j=0$（模式串指针，表示尚未匹配任何字符）
2. 当 $i \le len(T)$ 时循环：
   - 如果 $j > 0$ 且 $T[i] \neq S[j+1]$，则 $j = fail[j]$
   - 如果 $T[i] == S[j+1]$，则 `j++`
   - 如果 $j == len(S)$，说明找到了一个匹配，输出位置 $i - len(S) + 1$，然后 $j = fail[j]$ 继续寻找下一个匹配
   - `i++`

#### 代码实现
```cpp
#include<cstdio>
#include<algorithm>
#include<cstring>
using namespace std;
const int MAXN=1e6;
int n1,n2;
char s1[MAXN+5],s2[MAXN+5];
int fail[MAXN+5];
int main()
{
    scanf("%s",s1+1);
    scanf("%s",s2+1);
    n1=strlen(s1+1);
    n2=strlen(s2+1);
    for(int i=2;i<=n2;i++)//计算fail数组
    {
        int j=fail[i-1];
        while((s2[j+1]!=s2[i])&&(j>0))
        {
            j=fail[j];
        }
        if(s2[j+1]==s2[i])
        {
            j++;
        }
        fail[i]=j;
    }
    int p2=0;
    for(int i=1;i<=n1;i++)
    {
        while(s1[i]!=s2[p2+1]&&(p2>0))
        {
            p2=fail[p2];
        }
        if(s1[i]==s2[p2+1])
        {
            p2++;
            if(p2==n2)
            {
                printf("%d\n",i-n2+1);
                p2=fail[p2];
            }
        }
    }
    for(int i=1;i<=n2;i++)
    {
        printf("%d ",fail[i]);
    }   
}
```
> 注：代码中的fail数组直接存储border长度，与题目要求的输出一致。

### 例题：[Luogu P2375 【NOI2014】 动物园](https://www.luogu.com.cn/problem/P2375)

#### 题目描述
对于字符串 $S$ 的每个前缀 $i$，定义 $num[i]$ 为满足以下条件的字符串 $t$ 的数量：$t$ 既是前缀 $i$ 的真前缀，又是前缀 $i$ 的真后缀，且 $t$ 的长度不超过 $i/2$（即不重叠）。输出 $\prod_{i=1}^{n} (num[i]+1) \bmod 10^9+7$。

#### 思路
利用 KMP 的 border 递推关系。首先在求 fail 数组的同时，记录 $cnt[i]$ 表示前缀 $i$ 的 border 总数（包括所有嵌套的 border，含自身），递推式为 $cnt[i] = cnt[fail[i]] + 1$。

然后模拟 KMP 匹配过程，维护指针 $j$ 使其始终满足 $j \le i/2$（即不重叠），此时 $num[i] = cnt[j]$，最后累乘即可。

#### 代码实现
```cpp
#include<cstdio>
#include<cstring>
using namespace std;
const int MAXN=1e6, MOD=1e9+7;
char s[MAXN+5];
int fail[MAXN+5], cnt[MAXN+5], n;
int main()
{
    int T; scanf("%d", &T);
    while(T--)
    {
        scanf("%s", s+1);
        n=strlen(s+1);
        cnt[1]=1;
        for(int i=2, j=0; i<=n; i++)
        {
            while(j && s[j+1]!=s[i]) j=fail[j];
            if(s[j+1]==s[i]) j++;
            fail[i]=j;
            cnt[i]=cnt[j]+1;
        }
        long long ans=1;
        for(int i=2, j=0; i<=n; i++)
        {
            while(j && s[j+1]!=s[i]) j=fail[j];
            if(s[j+1]==s[i]) j++;
            while(j*2>i) j=fail[j];
            ans=ans*(cnt[j]+1)%MOD;
        }
        printf("%lld\n", ans);
    }
    return 0;
}
```

### 例题：[Luogu P3426 【POI2005】 SZA-Template](https://www.luogu.com.cn/problem/P3426)

#### 题目描述
给定一个字符串 $S$，你有一个印章，每次可以印出某个前缀 $t$（印章上的图案必须是 $S$ 的某个前缀）。印的时候可以重叠，即后一次印的图案可以覆盖前一次的图案。求能印出整个 $S$ 的最短的印章长度。

#### 思路
设 $f[i]$ 为印出前缀 $i$ 所需的最短印章长度。显然 $f[i]$ 要么等于 $i$（直接用整个前缀作为印章），要么等于 $f[fail[i]]$（沿用最短印章）。

关键判定：若最短印章 $f[fail[i]]$ 能覆盖到位置 $i-fail[i]$，则它也能顺带覆盖前缀 $i$。这是因为 $[1,fail[i]]$ 与 $(i-fail[i],i]$ 完全相同，只要这两个区间能通过印章拼接连上，整个 $[1,i]$ 就能被覆盖。用 $las[k]$ 记录印章 $k$ 当前能覆盖到的最远位置即可。

#### 代码实现
```cpp
#include<cstdio>
#include<cstring>
#include<algorithm>
using namespace std;
const int MAXN=5e5;
char s[MAXN+5];
int fail[MAXN+5], f[MAXN+5], las[MAXN+5], n;
int main()
{
    scanf("%s", s+1);
    n=strlen(s+1);
    for(int i=2, j=0; i<=n; i++)
    {
        while(j && s[j+1]!=s[i]) j=fail[j];
        if(s[j+1]==s[i]) j++;
        fail[i]=j;
    }
    for(int i=1; i<=n; i++)
    {
        f[i]=i;
        if(las[f[fail[i]]]>=i-fail[i]) f[i]=f[fail[i]];
        las[f[i]]=i;
    }
    printf("%d\n", f[n]);
    return 0;
}
```
